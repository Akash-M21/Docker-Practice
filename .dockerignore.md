# .dockerignore File

A `.dockerignore` file works like `.gitignore`, but for Docker builds.

It tells Docker which files or directories to exclude from the build context. These files are not sent to the Docker daemon during the image build process.

---

# Why .dockerignore is Important

- Reduces build context size  
- Speeds up Docker image builds  
- Prevents unnecessary files from being included in the image  
- Improves security by excluding sensitive files  

---

# How It Works

When you run:

```bash
docker build -t my-image .
```

Docker sends all files in the current directory (build context) to the Docker daemon.

A `.dockerignore` file filters out unwanted files before sending them.

---

# Example .dockerignore File

```dockerignore
node_modules
.git
.gitignore
Dockerfile
docker-compose.yml
*.log
*.env
dist/
target/
```

---

# Explanation

- `node_modules` → avoids heavy dependency folders  
- `.git` → excludes version control history  
- `*.log` → ignores log files  
- `.env` → prevents sensitive environment variables from leaking  
- `dist/`, `target/` → excludes build output folders  

---

# Benefits

✔ Faster Docker builds  
✔ Smaller images  
✔ Cleaner build context  
✔ Better security  

---

# Summary

- `.dockerignore` works like `.gitignore`  
- It excludes unnecessary files from Docker build context
